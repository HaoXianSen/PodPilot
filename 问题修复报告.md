### 3. ✅ 创建tag失败 - 已增强错误处理

**问题描述：**
- 创建tag失败时只显示"创建Tag失败"
- 没有详细的错误原因
- 没有解决建议

**解决方案：**
增强了`execute_create_tag`方法中的错误处理，提供详细的错误信息和解决建议

**具体修改：**

#### 3.1 推送失败处理
```python
if push_result.returncode == 0:
    self.log_message(f"Tag {tag_name} 已推送到远程仓库")
else:
    error_msg = push_result.stderr or "未知错误"
    self.log_message(f"⚠️ 推送Tag失败: {error_msg}")
    
    # 记录到历史（即使推送失败也记录）
    self.tag_history_manager.record_tag_operation(
        self.current_project,
        pod_name,
        'create',
        tag_name,
        {
            'pushed': False,
            'message': tag_message,
            'error': error_msg
        }
    )
```

#### 3.2 Git命令失败处理
```python
else:
    error_msg = result.stderr or "未知错误"
    self.log_message(f"❌ 创建Tag失败: {error_msg}")
    
    # 提供解决建议
    if "already exists" in error_msg:
        self.log_message("💡 提示: 该Tag已存在，请使用不同的名称或删除后重试")
    elif "not a git repository" in error_msg:
        self.log_message("💡 提示: 该目录不是Git仓库，请检查Pod本地路径")
    elif "Permission denied" in error_msg:
        self.log_message("💡 提示: 没有写权限，请检查目录权限")
    else:
        self.log_message("💡 提示: 请检查Git配置和网络连接")
```

#### 3.3 新增错误类型识别
- ✅ Tag已存在：提示使用不同名称或删除后重试
- ✅ 不是Git仓库：提示检查本地路径
- ✅ 权限问题：提示检查目录权限
- ✅ 网络问题：提示检查网络连接和Git配置

#### 3.4 历史记录增强

## 4. ✅ 批量创建Tag分支选择规则优化

**修改日期：** 2026-02-12

**修改文件：** `src/views/dialogs/batch_tag_dialog.py`

**修改内容：**
优化了分支选择的默认规则，现在会智能选择包含 "master" 的分支：

```python
# 默认优先选择包含"master"的分支（如 master, origin/master, feature/master-fix 等）
master_branches = [b for b in branches if "master" in b.lower()]
if master_branches:
    # 优先选择精确匹配 "master" 或 "origin/master"，其次是其他包含master的分支
    if "master" in master_branches:
        branch_combo.setCurrentText("master")
    elif "origin/master" in master_branches:
        branch_combo.setCurrentText("origin/master")
    else:
        branch_combo.setCurrentText(master_branches[0])
elif len(branches) > 0:
    branch_combo.setCurrentIndex(0)
```

**优先级顺序：**
1. **精确匹配** `"master"`
2. **精确匹配** `"origin/master"`  
3. **其他包含 master 的分支**（如 `feature/master-fix`、`dev-master` 等）
4. 如果没有包含 master 的分支，则选择列表中的第一个分支

**用户体验改进：**
- ✅ 自动优先选择主分支，减少用户手动选择操作
- ✅ 兼容各种 master 分支命名习惯
- ✅ 如果没有 master 分支，智能回退到第一个分支

## 功能说明

### 批量创建Tag中"保存当前Pod配置"说明

**保存配置的内容：**

在批量创建Tag对话框中，"保存当前Pod配置"按钮（save_btn）保存以下配置信息：

```python
self.pod_configs[self.current_row] = {
    "tag_name": tag_name,      # Tag名称（从表格第4列获取）
    "tag_message": tag_message, # Tag消息（从消息编辑框获取）
}
```

**配置存储位置：**
- 存储在 `self.pod_configs` 字典中
- 键为当前行的索引（row number）
- 值为包含 tag_name 和 tag_message 的字典

**触发时机：**
1. 手动点击"保存当前Pod配置"按钮时
2. 切换表格行时（`on_row_changed` 方法会自动调用）
3. 点击"批量创建所有Tag"按钮前（会自动保存当前行的配置）

**使用场景：**
- 用户在编辑某个Pod的Tag名称和消息后，点击保存按钮将配置暂存
- 切换到其他行编辑时，上一行的配置会自动保存
- 批量创建Tag时，会使用 `self.pod_configs` 中保存的配置信息

**注意：**
- 这个配置仅保存在内存中（`self.pod_configs` 字典）
- 对话框关闭后配置会丢失
- 实际的Tag创建操作会使用这些配置信息
