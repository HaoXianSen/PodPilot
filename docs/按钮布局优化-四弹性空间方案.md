# 按钮布局优化 - 三组件对称设计

## 问题历程

### 初始问题
删除"一键Branch"按钮后，右侧按钮没有与 pod 列表右边界对齐。

### 尝试过的方案

| 方案 | 结果 | 问题 |
|------|------|------|
| 移除 addStretch | ❌ | 右侧不对齐 |
| addStretch 在最后 | ❌ | Segmented 和退出开发距离太远 |
| addStretch 在中间 | ❌ | 右侧退出开发不对齐 |
| 硬编码计算宽度 | ❌ | 难以维护，扩展性差 |
| 左右 addStretch | ❌ | 分隔符不居中，空间不对称 |
| **四弹性空间方案** | ✅ | **完美解决！** |

---

## 最终方案：四弹性空间 + 三组件设计

### 🎯 设计理念

将按钮栏分为**三个独立组件**，组件之间用**分隔符 + 弹性空间**分隔：

```
[组件1]  ←space→  |  ←space→  [组件2]  ←space→  |  ←space→  [组件3]
```

### 📐 布局结构

```
pod_btn_layout (QHBoxLayout, spacing=0):
│
├── [组件1] left_btn_group (QWidget)
│   └── [配置 80px] [创建Tag 80px] [清理缓存 80px]
│
├── addStretch() ← 弹性空间1
├── separator1 | ← 分隔符1
├── addStretch() ← 弹性空间2
│
├── [组件2] segment_container (QWidget)
│   └── [开发模式|Branch模式|Tag模式] (Segmented Control)
│
├── addStretch() ← 弹性空间3
├── separator2 | ← 分隔符2
├── addStretch() ← 弹性空间4
│
└── [组件3] right_btn_group (QWidget)
    └── [退出开发 90px]
```

### ✨ 核心特性

#### 1. 四个弹性空间自动平均分配
Qt 的 `addStretch()` 会平均分配剩余空间：
- 弹性空间1 = 弹性空间2 = 弹性空间3 = 弹性空间4
- 分隔符1 自动居中在组件1和组件2之间
- 分隔符2 自动居中在组件2和组件3之间

#### 2. 三个独立组件
每个组件是一个 `QWidget`，包含自己的按钮和布局：
- **组件1**：功能按钮（配置、创建Tag、清理缓存）
- **组件2**：模式切换（Segmented Control）
- **组件3**：退出开发

#### 3. 自动对齐
- **左对齐**：组件1 自动靠左，与 pod_list 左边界对齐
- **右对齐**：组件3 自动靠右，与 pod_list 右边界对齐
- **整行填充**：pod_btn_layout 自动填充父容器宽度

### 🎨 视觉效果

```
[配置] [创建Tag] [清理缓存]    |    [开发|Branch|Tag]    |    [退出开发]
        ↑ 252px             相等  相等   ↑ 自动宽度      相等  相等  ↑ 90px
```

分隔符左右的空间完全相等，视觉对称。

## 代码实现

### 关键点

1. **pod_btn_layout.setSpacing(0)**
   - 关闭自动间距，完全由 addStretch() 控制

2. **四个 addStretch()**
   - 分隔符两侧各一个 → 分隔符居中
   - 两个分隔符左右各一个 → 整体居中

3. **组件封装**
   - 每个组件用 QWidget 包装
   - 组件内部的布局和间距独立管理

### 修改位置

**文件**：`src/views/main_window.py:315-422`

**核心改动**：
- 设置 `pod_btn_layout.setSpacing(0)`
- 添加四个 `addStretch()`
- 将按钮封装到三个 QWidget 组件中

## 优势对比

### 硬编码方案 ❌
```python
# 需要手动计算
segment_btn_width = 48  # = (240 - 90 - spacing) / 3
```

**缺点**：
- 添加/删除按钮需重新计算
- 修改间距需调整所有宽度
- 难以维护，容易出错

### 四弹性空间方案 ✅
```python
# 完全自动化
pod_btn_layout.addStretch()  # 自动分配
```

**优势**：
- ✅ 无需计算宽度
- ✅ 自动适应窗口大小
- ✅ 添加/删除按钮无需改动
- ✅ 分隔符自动居中
- ✅ 左右自动对齐

## 技术细节

### Qt 布局系统原理

**addStretch() 行为**：
- 多个 `addStretch()` 会平均分配剩余空间
- 例如：剩余空间 400px，4个 `addStretch()`，每个获得 100px

**QWidget 封装**：
- QWidget 会根据内部内容自动计算 sizeHint
- 使用 `setFixedSize()` 的按钮会固定宽度
- 使用 `setFixedHeight()` 的按钮会自动适应宽度

### 为什么需要四个 addStretch？

```
[组件1]  S1  |  S2  [组件2]  S3  |  S4  [组件3]
```

- **S1 和 S2**：让分隔符1居中在组件1和组件2之间
- **S3 和 S4**：让分隔符2居中在组件2和组件3之间
- **S1、S2、S3、S4 相等**：四个 addStretch() 平均分配

## 维护建议

### DO ✅
- 保持 `pod_btn_layout.setSpacing(0)`
- 保持四个 `addStretch()` 的位置
- 在组件内部修改按钮（添加/删除/调整）

### DON'T ❌
- 不要删除任何 `addStretch()`
- 不要修改 `pod_btn_layout.setSpacing(0)`
- 不要在组件之间添加额外的 spacing
