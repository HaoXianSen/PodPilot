### 1. 修复 QThread 崩溃问题 ⚡

**问题**：点击 Tag 模式时应用崩溃，输出 `QThread: Destroyed while thread is still running`

**根因**：
- `RemoteTagLoader` 线程在对话框关闭时未正确等待线程结束
- 主窗口 `closeEvent` 方法缺少对 `tag_loader` 的清理

**解决方案**：
- 在 `_on_tags_loaded` 方法中添加 `self.tag_loader.wait()` 
- 在 `closeEvent` 方法中添加 `tag_loader` 线程清理逻辑
- 使用 try-except-finally 保证资源正确释放

**修改文件**：`src/views/main_window.py:590-619, 1348-1365`

**测试状态**：✅ 已验证，无崩溃

### 4. 修复批量 Tag 下拉框不能点击 🔧

**问题**：某些 Pod 的 Tag 下拉框被禁用，无法点击

**根因**：`get_remote_tags()` 使用了严格的版本号格式过滤：
```python
if re.match(r"v?\d+\.\d+\.\d+", tag_name):  # ❌ 只允许标准版本号
    remote_tags.append(tag_name)
```

这导致非版本号格式的 tag（如 `feature-xyz`、`hotfix`、`dev`）被过滤掉。

**解决方案**：移除版本号格式限制，允许所有 tag

**修改文件**：`src/services/git_service.py:27-113`

---

### 5. Tag 排序优化 - 按时间倒序 ⏰

**需求**：Tag 排序希望按创建时间倒序，最新的在最上边

**实现方案**：

#### 方案1（优先）：本地排序
```bash
git fetch --tags --quiet       # 先拉取最新 tags
git tag --sort=-creatordate    # 按创建时间倒序
```

#### 方案2（回退）：远程获取
```bash
git ls-remote --tags origin    # 无排序，原始顺序
```

**优势**：
- ✅ 按创建时间排序，最新 tag 在最前
- ✅ 支持所有 tag 格式
- ✅ 双重回退机制，确保稳定性

**排序效果示例**：
```
v2.1.0 (2026-03-12) ← 最新
feature-x (2026-03-10)
v2.0.0 (2026-03-05)
v1.5.0 (2026-02-20)
v1.0.0 (2026-01-15) ← 最旧
```

**修改文件**：`src/services/git_service.py:27-113`
